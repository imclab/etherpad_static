P2PU Platform

One overall platform strategy with three equally important goals:
  * our key community / key users are happy and active
  * we are more agile
  * there is more experimentation in the context of P2PU


(1) core > develop and support core set of services

goals:

  * define a core set services we support (e.g. courses, profiles, badges, mentoring ...)
  * design each as a /narrowly/ defined product
  * build competitive high quality offerings for each 

buckets:

  * design features, review implementation (less technology management)
  * lead projects with partners (school of data)


(2) lab > define architecture & processes to spur innovation

goals:

  * more experimentation by us and others
  * three levels of experimentation: API, modules, embeds
  * strategy for capturing value of experimentation (esp. those we invest resources into)
  * more rapidly developed prototypes / partnerships
  * process for successful experiments to move from edges into core

two buckets:

  * build stackmaps for education
  * help others build stackmaps for education
